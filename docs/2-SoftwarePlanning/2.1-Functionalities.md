# 2.1 Функціональні та нефункціональні вимоги

## Функціональні вимоги (Functional Requirements)
1. **FR-01:** Система повинна дозволяти користувачу створювати профіль із зазначенням ваги та рівня фізичної активності.
2. **FR-02:** Система повинна розраховувати денну норму води на основі введених антропометричних даних.
3. **FR-03:** Система повинна забезпечувати можливість внесення об'єму спожитої рідини (у мл).
4. **FR-04:** Система повинна відображати історію споживання за останні 7 днів.

## Нефункціональні вимоги (Non-functional Requirements)
1. **NR-01 (Продуктивність):** Час відгуку системи на додавання запису не повинен перевищувати 0.5 секунди.
2. **NR-02 (Надійність):** Система повинна зберігати дані користувача локально, щоб вони не зникали при відсутності інтернету.
3. **NR-03 (Юзабіліті):** Додавання нового запису про воду має виконуватися не більше ніж за 3 кліки з головного екрана.
