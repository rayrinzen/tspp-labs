## Виконані етапи 
### Лабораторна робота №1
1.  **Аналіз потреб:** Визначено об'єкт дослідження (фізіологічна потреба у воді).
2.  **Модель DIKW:** Описано перехід від сирих даних до мудрості керування здоров'ям.
3.  **Vision:** Сформовано бачення продукту та його переваги над конкурентами.
4.  **Use Case:** Спроектовано діаграму прецедентів для основних дій користувача.
5.  **Gherkin:** Описано поведінковий сценарій додавання води у форматі BDD (Behavior Driven Development).

### Лабораторна робота №2: Планування
1. **Аналіз вимог:** Визначено функціональні та нефункціональні вимоги згідно з ISO.
2. **GitHub Projects:** Створено Kanban-дошку для керування завданнями проєкту.
3. **Керування версіями:** Реалізовано робочий процес через Pull Requests.

### Лабораторна робота №3: Проектування даних
1. **UML-діаграма:** Спроектовано концептуальну модель класів застосунку за допомогою PlantUML.
2. **JSON Schema:** Розроблено специфікацію для логічної структури даних записів споживання води.

### Лабораторна робота №4: Деталізація даних та CI
1. **SQL-скрипт:** Розроблено деталізований опис логічної моделі даних згідно з нормами SQL Coding Conventions.
2. **GitHub Actions:** Налаштовано автоматизовану перевірку (Linting) SQL-коду за допомогою утиліти `sqlfluff`.
3. **Continuous Integration:** Реалізовано робочий процес автоматичної валідації коду при кожному push або PR.

### Лабораторна робота №5: Проєктування класів та тестування
1. **Прототип програмних класів:** На основі концептуальної моделі розроблено діаграму класів (PlantUML) з урахуванням типів даних, методів та рівнів видимості.
2. **Тестові сценарії:** Розроблено сценарії верифікації вимог за принципом "Black Box" для перевірки функціональності (додавання записів) та надійності (валідація даних).
3. **Верифікація вимог:** Сформовано чіткі алгоритми перевірки відповідності реалізації початковим вимогам проєкту.

### Лабораторна робота №6: Контейнеризація (Docker)
1. **Dockerfile:** Розроблено інструкції для створення імейджу бази даних на основі PostgreSQL.
2. **Docker Compose:** Налаштовано конфігурацію для автоматичного розгортання середовища та прокидання портів.
3. **Автоматизація:** Реалізовано копіювання SQL-схеми з попередніх робіт для автоматичної ініціалізації БД при запуску контейнера.
